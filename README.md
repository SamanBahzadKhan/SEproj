## Project Overview

**Project Name:** Counseling Appointment System   
**Course:** CS360  
**Group Name:** Fridge  
**Group Number:** 30

**Concept:** An appointment management platform that connects students with counseling services, enabling easy booking, scheduling, and session management.

**Primary Users:** Students, Counselors, Counseling Office Administrators

**Key Challenges:** Privacy and confidentiality, scheduling conflict management, reliable notification system, no-show prevention

### Core Features / User Stories

#### Phase 1 — Core (Priority)
- [ ] Counselor discovery and availability viewing
- [ ] Time slot booking system
- [ ] Appointment reminders and notifications
- [ ] Rescheduling and cancellation functionality
- [ ] Counselor availability management dashboard

#### Phase 2 — Progressive
- [ ] No-show tracking and analytics
- [ ] Pre-session intake forms
- [ ] Post-session feedback mechanism
- [ ] Appointment history for students and counselors

> **Scope Note:** Focus on appointment booking flow, availability management, and basic notifications. Intake forms and feedback to be added progressively.

# 4. Storyboards

Storyboards describe interaction scenarios that demonstrate how users interact with the system to perform common tasks.

Location in repository:

```
docs/storyboard.md
```

Implemented scenarios:

1. Authentication / Login
2. Student Books Appointment
3. Student Cancels or Reschedules Appointment
4. Counselor Manages Availability

Each storyboard describes:

* reference to the screen involved
* user actions
* system responses
